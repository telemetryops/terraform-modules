# CI/CD Powered Repository

Generated by the menu tool. All builds and deployments are handled by GitHub Actions.

## Development Principles

**NO BACKWARD COMPATIBILITY**: Remove deprecated code immediately. Clean breaks over gradual migration. If it's not used, delete it.

**NO FALLBACKS OR HARDCODING**: If a dependency is unavailable, FAIL IMMEDIATELY with clear error. If configuration is missing, FAIL IMMEDIATELY - do not guess defaults. Make failures loud and obvious - silent fallbacks hide real problems.

## Menu Tool Usage

**IMPORTANT:** `menu` is a global command. NEVER run with `python` or any Python prefix.

Add services to this repository:

```bash
menu add-service \
  --name <service-name> \
  --language <python|rust> \
  --repo <this-repo-name> \
  --api-gateway  # Optional: adds API Gateway integration
```

## AWS Profiles

**CRITICAL: No SSO.** All local AWS profiles use IAM user credentials.

```bash
aws configure list-profiles  # List profiles
AWS_PROFILE=x aws sts get-caller-identity  # Check auth
```

## Git Commands (Makefile)

```bash
make commit MSG="your commit message"   # Add, commit, and push all changes
make amend MSG="new message"            # Amend last commit and force push
make push                               # Push current branch
```

## Deployment

Deployments happen automatically via GitHub Actions:
- Push to feature/* → deploys to dev account
- Push/merge to dev → deploys to dev account
- Push/merge to staging → deploys to staging account
- Push/merge to prod → deploys to prod account

## Infrastructure Patterns

S3 State Buckets: {repo-name}-{account-id}
DynamoDB Lock Tables: {repo-name}-{account-id}
ECR Repositories: {service-name}-lambda-repo

## Terraform Modules

Pre-built, maintained modules in `terraform/modules/`:
- `lambda_container_function` - Python Lambda with Docker + ECR
- `lambda_rust_function` - Rust Lambda with cargo-lambda
- `api_gateway_endpoints` - API Gateway configuration

These modules are kept up-to-date and production-ready. Use them for consistent infrastructure.

## Backend Microservice Architecture (if applicable)

Services are automatically discovered from src/ directory:
```
src/
├── user-service/
│   └── lambda_function.py (Python)
├── auth-service/
│   └── src/main.rs (Rust)
```

Python services use Docker + ECR.
Rust services use cargo-lambda + direct Lambda deployment.

### API Gateway Integration

Use `--api-gateway` flag when adding services. Creates REST API with:
- CORS defaults (origin `'*'`)
- `/health` endpoint (GET)
- Terraform outputs for API Gateway URL

Add routes by editing `{service}_endpoints` local in `terraform/lambda_{service}.tf`.
